# Objetivo de calidad

Establecer criterios y actividades para garantizar que la documentación y la propuesta técnica de KaiVoxx FG sean consistentes, comprensibles y verificables.

# Actividades de calidad

| ID | Actividad | Propósito |
| --- | --- | --- |
| QA-01 | Revisión de requerimientos | Confirmar que RF y RNF sean completos, no ambiguos y medibles. |
| QA-02 | Validación de arquitectura | Verificar coherencia entre contexto, contenedores, componentes y decisiones. |
| QA-03 | Pruebas funcionales | Comprobar flujos de registro, análisis, alerta, revisión y bloqueo. |
| QA-04 | Pruebas no funcionales | Evaluar rendimiento, seguridad, trazabilidad y disponibilidad esperada. |
| QA-05 | Revisión de documentos | Aplicar una lista de chequeo para detectar omisiones o inconsistencias. |

# Métricas de revisión

- Consistencia entre problema, objetivos, alcance y arquitectura.

- Completitud de RF, RNF, casos de uso y riesgos.

- Claridad visual de tablas, diagramas y presentación final.

# 1. Enfoque metodológico

Se propone una ejecución por entregables, con una lógica iterativa inspirada en Scrum para priorizar artefactos y revisar avances por bloques.

# 2. Roles sugeridos

| Rol | Responsabilidad |
| --- | --- |
| Líder del proyecto | Coordinar entregables, fechas y coherencia general. |
| Analista funcional | Redactar requerimientos, casos de uso e historias. |
| Arquitecto | Definir C4, decisiones técnicas y documentación de diseño. |
| Gestor de calidad | Revisar consistencia, formato y criterios de aceptación. |

# 3. Gestión de alcance y cambios

- Todo cambio debe quedar documentado en una versión del artefacto.

- La prioridad es conservar coherencia entre problema, solución y arquitectura.

- Los cambios deben ser aprobados por el responsable académico o el grupo.

# 4. Comunicación

- Reuniones cortas de seguimiento.

- Documento central con versiones y observaciones.

- Presentación de avances por hito principal.

# 5. Calidad, riesgos y control

- Aplicar checklist de revisión a cada documento.

- Mantener matriz de riesgos viva durante el desarrollo.

- Validar que los diagramas se relacionen con los requerimientos.

| ID | Historia | Prioridad | Criterio de aceptación |
| --- | --- | --- | --- |
| US-01 | Como usuario de contabilidad, quiero ver una lista priorizada de alertas para revisar primero los casos más críticos. | Alta | La lista se ordena por score, fecha y criticidad. |
| US-02 | Como tesorero, quiero retener pagos sospechosos antes de ejecutarlos para evitar pérdidas. | Alta | El pago cambia a estado retenido y queda evidencia. |
| US-03 | Como auditor, quiero consultar el historial completo de un caso para verificar quién tomó cada decisión. | Alta | El caso muestra línea de tiempo y trazabilidad. |
| US-04 | Como administrador, quiero configurar reglas de fraude para ajustar el sistema al contexto de la empresa. | Alta | Las reglas se guardan y quedan activas de inmediato o con aprobación. |
| US-05 | Como usuario de gerencia, quiero ver indicadores del sistema para medir ahorro y control. | Media | Existe tablero con KPIs clave. |
| US-06 | Como analista, quiero adjuntar observaciones al revisar una alerta para documentar la decisión. | Media | Se guardan comentarios y evidencias. |

# Valor ágil

- Las historias priorizan experiencia operativa y toma de decisiones.

- Cada historia se puede convertir en tareas de desarrollo, pruebas y documentación.

- El backlog debe revisarse con usuarios clave para afinar el alcance.
